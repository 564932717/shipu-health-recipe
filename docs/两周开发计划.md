# 个性化健康食谱规划系统两周开发计划

## 项目目标

在两周内完成一个可演示的鸿蒙 + SpringBoot 个性化健康食谱规划系统，形成“健康档案录入、健康评估、三餐推荐、饮食记录、营养统计、健康周报”的完整闭环。

## 人员分工

| 角色 | 人数 | 主要职责 |
|---|---:|---|
| 组长 / 文档 / 答辩 | 1 | 需求梳理、进度跟踪、文档整理、PPT 制作、答辩汇报 |
| 鸿蒙前端 A | 1 | 首页、登录注册、健康档案、个人中心 |
| 鸿蒙前端 B | 1 | 三餐推荐、食谱详情、饮食记录、统计图表、原子化卡片 |
| 后端 A | 1 | SpringBoot 工程、用户接口、健康档案接口、食谱接口 |
| 后端 B / 算法测试 | 1 | 健康评估、推荐算法、周报接口、接口测试、测试报告 |

## 10 个工作日计划

| 天数 | 开发内容 | 交付物 |
|---|---|---|
| 第 1 天 | 明确需求、功能范围、系统边界、任务分工 | 需求清单、分工表 |
| 第 2 天 | 页面原型、数据库设计、接口设计 | 原型图、ER 图、接口文档 |
| 第 3 天 | 前后端工程搭建、模拟数据准备 | 可运行工程骨架 |
| 第 4 天 | 用户登录注册、健康档案录入 | 用户与档案模块 |
| 第 5 天 | BMI、推荐热量、营养比例计算 | 健康评估模块 |
| 第 6 天 | 食谱库、搜索、详情、收藏 | 食谱基础模块 |
| 第 7 天 | 三餐推荐、忌口过滤、换一换 | 推荐闭环 |
| 第 8 天 | 饮食记录、每日摄入统计、图表展示 | 记录与统计模块 |
| 第 9 天 | 健康周报、AI 建议、原子化卡片 | 周报与特色能力 |
| 第 10 天 | 联调测试、Bug 修复、文档和 PPT | 可答辩版本 |

## 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| HarmonyOS 环境配置耗时 | 前端开发延迟 | 先用 mock 数据完成页面，再统一联调 |
| 数据库安装不稳定 | 后端接口无法演示 | 第一版使用内存模拟数据，后续替换 MySQL |
| 推荐算法过复杂 | 两周内难以完成 | 先实现规则推荐，再扩展个性化权重 |
| 前后端接口变动 | 联调成本增加 | 先固定接口文档，新增字段保持兼容 |
| PPT 和文档挤压开发时间 | 答辩材料不足 | 每日同步文档，最后一天只做整理和演示修复 |
